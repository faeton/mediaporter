## Architecture

ATC is the **control channel** (metadata, sync orchestration). AFC is the **data channel** (actual file bytes).

### High-Level API Flow (via ATHostConnection)

```
Host                                              Device
 |                                                  |
 |-- ATHostConnectionCreateWithLibrary ------------>|
 |-- SendPowerAssertion(true) --------------------->|
 |-- SendHostInfo (Version, LibraryID, hostname) -->|
 |                                                  |
 |<--- InstalledAssets (app list) -------------------|
 |<--- AssetMetrics (storage stats) ----------------|
 |<--- SyncAllowed (permissions) -------------------|
 |                                                  |
 |-- SendMessage(RequestingSync) ------------------>|
 |   { Dataclasses: [Media, Keybag],               |
 |     DataclassAnchors: {Media: "0"},              |
 |     HostInfo: {Grappa: <84 bytes>, ...} }        |
 |                                                  |
 |<--- SyncAllowed ----------------------------------|
 |<--- InstalledAssets ------------------------------|
 |<--- ReadyForSync --------------------------------|
 |     { DataclassAnchors: {Media: "<syncNum>"},    |
 |       DeviceInfo: {Grappa: <83 bytes>} }         |
 |                                                  |
 |== AFC: Write binary sync plist + CIG ============|
 |   /iTunes_Control/Sync/Media/Sync_XXXX.plist    |
 |   /iTunes_Control/Sync/Media/Sync_XXXX.plist.cig|
 |   (plist must have is_movie:True + location.kind)|
 |                                                  |
 |-- SendPowerAssertion(true) --------------------->|
 |-- FinishedSyncingMetadata ---------------------->|
 |   { SyncTypes: {Keybag:1, Media:1},             |
 |     DataclassAnchors: {Media: "<syncNum>"} }     |
 |                                                  |
 |<--- AssetManifest (AssetType=Movie) -------------|
 |                                                  |
 |== AFC: Upload to /iTunes_Control/Music/Fxx/ =====|
 |   (single upload — Airlock NOT needed)           |
 |-- FileBegin ------------------------------------>|
 |   { AssetID: "<string>", FileSize: int,          |
 |     TotalSize: int, Dataclass: "Media" }         |
 |-- FileProgress (repeated) ---------------------->|
 |-- FileComplete --------------------------------->|
 |   { AssetID: "<string>",                         |
 |     AssetPath: "/iTunes_Control/Music/Fxx/...",  |
 |     Dataclass: "Media" }                         |
 |                                                  |
 |<--- SyncFinished --------------------------------|
```

### Wire Command Name Mapping

The high-level ATHostConnection API uses different names than what appears on the wire:

| High-level API call | Wire command name |
|---------------------|-------------------|
| ATHostConnectionSendMetadataSyncFinished | `FinishedSyncingMetadata` |
| ATHostConnectionSendFileBegin | `FileBegin` |
| ATHostConnectionSendFileProgress | `FileProgress` |
| ATHostConnectionSendAssetCompleted | `FileComplete` |

**CRITICAL**: Using the wrong command name (e.g., "MetadataSyncFinished" instead of "FinishedSyncingMetadata") causes the device to silently ignore the message.

### All Values Are STRINGS on the Wire

Anchors, AssetIDs, and other numeric-looking values are **strings** in the wire plist format:
- DataclassAnchors: `{Media = "12"}` not `{Media = 12}`
- AssetID: `"349645419467270165"` not integer
- Passing integers instead of strings causes silent failures

## Grappa Authentication

### Host Grappa (84 bytes, host → device)

- Sent in `RequestingSync.Params.HostInfo.Grappa`
- Format: `0x0101` (version) + `0x11` × 16 (CIG engine init state) + 66 bytes crypto
- **Can be hardcoded/replayed** — go-tunes uses a static blob, we proved replay works
- Generated by `AirFairSyncGrappaCreate` in AirTrafficHost.framework (requires CoreFP/fairplayd entitlement for dynamic generation)
- Same blob found in `yinyajiang/go-tunes` `proto.go` as `deviceGrapa` constant

### Device Grappa (83 bytes, device → host)

- Received in `ReadyForSync.Params.DeviceInfo.Grappa`
- Used as key for CIG signature computation
- Different each session (nonce-based)

### Error Codes

| Code | Meaning | Cause |
|------|---------|-------|
| 12 | No Grappa | `RequestingSync` sent without `HostInfo.Grappa` field |
| 4 | Invalid Grappa | Grappa blob present but contents invalid (e.g., all 0x11) |
| 23 | Wrong sequence | `MetadataSyncFinished` sent before `RequestingSync` |

## CIG (Cryptographic Integrity Guarantee)

### Purpose

Signs sync plists so the device can verify they weren't tampered with.

### Algorithm

- Modified SHA-1 with custom S-box key expansion
- Input: device Grappa (83 bytes) + plist raw bytes
- Output: 21 bytes (1-byte prefix + 20-byte HMAC-SHA1-like digest)
- Implementation: `cig.cpp` in go-tunes (10K+ lines, from publicly available source)

### C function signature

```c
BOOL cigCalc(byte* pGrappa, byte* pData, int iDataLen, byte* pCig, int* pCigLen)
```

### When CIG is needed

- **Required**: `/iTunes_Control/Sync/Media/Sync_XXXXXXXX.plist` → `.cig` sidecar
- **Not required**: Dataclass-specific plist (e.g., `/iTunes_Control/Music/Sync/Sync_XXXXXXXX.plist`)

## Sync Plists (Written via AFC before MetadataSyncFinished)

**Format**: Binary plist (FMT_BINARY), NOT XML.

CIG sidecar: same path + `.cig` (21 bytes computed from device Grappa + plist bytes).

**Critical fields for video**:

**Dead end fields** (all ignored by device): `media_kind`, `media_type`, `mediaKind`, `mediaType` in any dict location.

## Keybag Parameter

"Keybag" is a **dataclass name string**, NOT a crypto keybag:

- For ringtones: `"Ringtone"`
- For video/music: `"Media"`
- Always paired with literal `"Keybag"` in the `Dataclasses` array: `["Media", "Keybag"]`

## AssetID / PID

- Random `uint64` (not sequential)
- Generated by host for new tracks
- Used to correlate FileBegin/FileProgress/AssetCompleted
- Also used as `pid` in sync plist `insert_track` operations

## File Paths

- **Final media files**: `/iTunes_Control/Music/F{00-49}/{4-char-random}.mp4` (playback path, used in FileComplete — single upload target)
- ~~**Airlock staging**~~: `/Airlock/Media/<AssetID>` — NOT needed. `is_movie: True` in plist sets media_type correctly without Airlock.
- **Sync plists**: `/iTunes_Control/Sync/Media/Sync_{syncNum:08d}.plist` + `.cig`
- ~~Dataclass plists: `/iTunes_Control/{Dataclass}/Sync/...`~~ — NOT needed, dead end

## Experimental Results (2026-04-06) — COMPLETE WORKING SOLUTION

### Complete Working Sync Flow (FINAL)

Video sync to iPad TV app is fully working from Python. Final implementation: `scripts/atc_nodeps_sync.py` (zero-dependency, pure ctypes + Apple frameworks).

1. ATC handshake with replayed 84-byte Grappa blob → ReadyForSync
2. AFC: write **binary plist** (with `is_movie: True` + `location.kind`) + CIG to `/iTunes_Control/Sync/Media/`
3. ATC: `SendPowerAssertion(conn, true)` ← REQUIRED
4. ATC: `FinishedSyncingMetadata` with anchor as STRING ← REQUIRED
5. Device: `AssetManifest` (AssetType=**Movie**, lists files to transfer)
6. AFC: upload file to `/iTunes_Control/Music/Fxx/name.mp4` (single upload — no Airlock needed)
7. ATC: `FileBegin` with final path
9. ATC: `FileComplete` with FINAL path
10. ATC: `FileError` for stale pending assets
11. Device: `SyncFinished` → entry appears in TV app

### Three Breakthroughs (2026-04-06)

Discovered via LLDB AFC tracing (breakpoints on AFCFileRefWrite):

1. **`is_movie: True`** in the item dict — this is THE key field that triggers media_type=2048.
2. **`location.kind: "MPEG-4 video file"`** — file type descriptor in the location dict, sets location_kind_id=4.
3. ~~`/Airlock/Media/<AssetID>` staging path~~ — NOT needed. Previously thought essential, but confirmed 2026-04-06 that `is_movie: True` alone sets media_type correctly. Single upload to final path works.

### DB Entries Created Successfully
